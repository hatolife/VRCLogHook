# VRC LogHook 仕様メモ（ドラフト）

## 1. プロジェクト概要
- プロジェクト名: `VRC LogHook`
- 目的: VRChatのローカルログを監視し、ユーザー定義条件に合致した新規追記を外部通知する
- 公開予定: GitHub
- ライセンス: CC0
- 実行ファイル名:
  - `vrc-loghook`（core/CLI）
  - `vrc-loghook-gui`（GUI）

## 2. 基本方針
- VRChat公式APIは使用しない
- VRChatアカウントのログイン機能（ID/パスワード等）は実装しない
- ローカルログ監視のみで完結
- 最優先ポリシー: 透明性が高いこと / 信頼性が高いこと / セキュリティが高いこと / 負荷が少ないこと

## 3. 対応環境と配布
- 対応OS: Windows / macOS / Linux
- 実装言語: Go（コンパイル言語）
- 配布形態: 各コンポーネントを単体実行ファイルとして配布する（`vrc-loghook` / `vrc-loghook-gui`）

## 4. 起動・常駐モード
- GUI起動時: GUIアプリケーションとして常駐
- コマンド起動時: GUIなしで常駐
- コマンド起動からGUIを開く手段を提供
- 重視点: 信頼性・セキュリティ・低常駐コスト
- 実行ファイルはCLIコアとGUIを分離する
- CLIコア（`vrc-loghook`）が監視/判定/通知/設定反映を担う
- GUI（`vrc-loghook-gui`）はクライアントとして動作し、将来的に別実装へ置換可能にする
- GUIとCLIコアの連携仕様（ローカルAPI/IPC）は公開し、互換性を重視する
- リポジトリ構成は `core/` と `gui/` を分離する

## 5. ログ監視仕様
- 監視頻度目安: 1分以内（設定変更可能）
- 監視対象パス: 設定変更可能
- 監視対象パスの初期値: VRChat公式既定値を使用
- 起動時のみ既存行をチェック
- 起動後は追加行のみを対象
- ログローテーション（新しいログファイルへの切り替え）に自動追従
- オフセット永続化を有効時は、前回位置から再開を優先し、既存行の再チェックは行わない
- オフセット情報がない初回起動時のみ、起動時既存行チェック方針を適用する

## 6. マッチングと通知
- マッチ処理は軽量寄りで実装
- ユーザーがHookを新規定義可能
- 通知先はDiscord等（初期はDiscord Webhookを優先）
- Hookは上級者向けのオプション機能として扱う
- 危険性を明示し、同意したユーザーのみ利用可能にする（GUIでは初期非表示を前提）
- 追加通知チャネルは需要ベースで拡張する
- 当面はWebhook通知 + コマンド実行Hookで拡張性を担保する

## 7. 設定システム
- 各種値はユーザーが変更可能
- 設定形式: 初版からJSON/HJSONの両対応
- コメント記述可能な運用を重視
- 設定ファイルが存在しない場合は初期値で自動生成
- 4/5/6の挙動に関する値（再試行、重複判定、性能上限など）は設定値として保持
- 初期値は実用的なデフォルトを採用
- ホットリロード対応
- 設定ファイルやドキュメント等の出力物はUTF-8
- 設定値は透明性のため、余程の理由がない限り公開・調整可能にする
- ホットリロード可否は設定項目ごとに定義する
- GUIでは再起動が必要な設定項目名に `*` を付けて明示する
- ホットリロード対象（既定）: 監視頻度、マッチ条件、通知先、再試行設定、重複判定設定
- 再起動必須対象（既定）: IPC関連設定、実行モード関連設定、ログ保存先の一部低レベル設定

## 8. 透明性・信頼性要件（必須）
以下を必須機能として実装する。
- `--dry-run`
- 通知送信前/代替のローカル出力
- 監視状態表示
- 自己ログ保存
- 機密情報マスキング（Webhook URL等）
- 設定スキーマ検証
- 送信先・送信内容の明確化
- Hook実行の安全設計（既定無効、明示有効化、タイムアウト、並列上限）

追加で、可能な限り以下のセキュリティ施策を取り込む。
- 設定ファイル権限チェック
- 依存脆弱性チェック、ハッシュ公開など配布の信頼性担保
- セキュリティ関連ドキュメントの継続更新（既知リスク、運用注意）

## 11. 運用・リリース方針

## 12. サポート方針
- Windowsを主対象として優先対応
- macOS / Linuxは動作対応を行う

## 13. デフォルト設計判断（ポリシー反映）
未確定項目は、上記ポリシーに従って以下を初期採用とする。

- GUI-CLI通信: `localhost` ではなくOSローカルIPC（WindowsはNamed Pipe、macOS/LinuxはUnix Domain Socket）を優先
- GUI-CLI認証: ローカル接続でもセッショントークンを要求
- ログ監視: 低負荷ポーリングを基本に、ログ切替を安全に追従
- 状態永続化: 読み取りオフセットを保存し、再起動時の重複通知を抑制
- 通知失敗時: 指数バックオフ + 上限付き再試行
- Hook機能: 既定無効、明示同意で有効化、タイムアウト/並列上限を強制
- 互換性: 設定フォーマットとGUI-CLI連携仕様は後方互換を重視
- GUI-CLI連携仕様: バージョン番号を持ち、互換性ポリシー（同一メジャー互換）を採用

## 14. 実装詳細の固定方針
- GUI技術: 初版GUIは軽量・配布容易性を重視し、置換可能な薄いクライアントとして実装する
- IPC仕様:
  - Windows: Named Pipe
  - macOS/Linux: Unix Domain Socket
  - メッセージ形式: UTF-8 JSON
  - すべての要求でセッショントークン認証を必須
- API仕様:
  - `v1` のメジャーバージョンを定義し、同一メジャー内で後方互換を維持
  - 主要機能（状態取得、開始/停止、設定読取/更新、GUI起動）を公開
- 通知ペイロード:
  - Discord向けに安全な既定テンプレートを用意
  - 機密情報を含む可能性のある値は送信前にマスク可能とする
  - 本文サイズ上限を設定可能にする
- Hook I/O契約:
  - 引数は構造化された固定順序で渡す
  - 標準入力にイベントJSONを渡すモードを提供
  - 終了コード0を成功、それ以外を失敗として扱う
  - タイムアウト超過時は強制終了し失敗扱い
- 状態永続化:
  - 監視ファイル単位で読み取りオフセットと最終更新時刻を保持
  - 定期保存 + 正常終了時保存
  - 破損時は安全側で復旧（初回起動相当）し、警告を出す
- 設定スキーマ:
  - 型検証、範囲検証、未知キー扱いを定義
  - すべての公開設定にデフォルト値を持たせる
  - 既定で負荷と安全性を優先した値を採用
