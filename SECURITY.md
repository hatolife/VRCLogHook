# セキュリティポリシー

## 対象範囲
VRC LogHook は、ローカルログ監視と任意の通知 / Hook 機能を対象とします。

## ライセンスと責任範囲
CC0 1.0（Public Domain Dedication）:  
https://creativecommons.org/publicdomain/zero/1.0/deed.ja

本プロジェクトは、CC0の方針に基づき自由に利用できます。  
一方で、法的に許される最大限の範囲で、作者は明示・黙示を問わず保証や責任を負いません。

## 安全利用条件（推奨）
以下を満たす運用を「安全利用」の基準とします。
1. GitHub Release から配布バイナリ（`vrc-loghook` / `vrc-loghook-gui`）のみ取得する
2. `SHA256SUMS` でハッシュ照合する
3. 配布バイナリを無改変で同一ディレクトリに配置して使用する
4. `hooks.enabled=false`（既定値）のまま利用する
5. `notify.discord.webhook_url` は自分が管理する送信先のみ設定する
6. 設定ファイルと `ipc.token` のアクセス権を本人用途に限定する

この条件下で、実用上のリスクを低く抑えた利用が可能です。

## セキュリティ原則
- APIレス: VRChat 公式 API は使用しない
- ログインレス: VRChat アカウント資格情報は扱わない
- 露出最小化: 機密値はログ / ステータス出力でマスク
- セーフデフォルト: 危険な Hook 実行は既定で無効

## Hookのリスク
Hook 実行は任意コマンドを実行できます。
- 既定値: 無効
- `unsafe_consent` の明示同意が必要
- タイムアウトと並列数上限を設定可能

## 利用者が注意すべき事項
1. すべきこと: GitHub Release から取得し、`SHA256SUMS` で照合してから実行する  
   しない場合: 改ざん済みバイナリを実行するリスクが上がる
2. すべきでないこと: 出所不明の配布物・第三者が再配布した実行ファイルを使う  
   起こりうること: 悪性プログラム実行、Webhookやログ情報の漏えい
3. すべきこと: `hooks.enabled=false` のまま使う（必要がない限り有効化しない）  
   有効化した場合に起こりうること: 任意コマンド実行により、ファイル破壊や情報送信
4. すべきこと: Webhook URL を秘密情報として扱い、漏えい時は再生成（ローテーション）する  
   漏えい時に起こりうること: 第三者による不正投稿、通知チャンネル汚染
5. すべきこと: 設定ファイルと `ipc.token` を本人以外が読めない権限で管理する  
   しない場合: 同一端末の他ユーザー/プロセスから設定・IPC操作されるリスク
6. すべきでないこと: 共有端末・共用アカウントでそのまま運用する  
   起こりうること: 設定改変、Webhook流出、意図しない停止/再読込
7. すべきこと: OSとセキュリティソフトを更新し、端末のマルウェア対策を行う  
   前提が崩れた場合: 本ツール単体では安全を保証できない

## 脆弱性報告
脆弱性は、再現手順を最小構成で添えてリポジトリの Security Advisory または Issue で報告してください。

## 現在のハードニング状況
実装済み:
- 設定バリデーション（未知キー拒否を含む）
- Webhook / Token のマスキング
- `--dry-run` モード
- ローカルイベントログ出力
- 上限付き指数バックオフによる再試行
- Hook 安全制御（明示同意・タイムアウト・並列上限）
- Windows Named Pipe / Unix Domain Socket IPC（トークン認証）
- IPCトークンの起動時再生成（`<config-dir>/ipc.token`、設定ファイル固定値に非依存）
- GUI起動前の識別チェック（`--identity`）
- リリースビルドでのGUIハッシュ埋め込みと起動時不一致警告（デフォルトON）

## GUI差し替えリスクへの対応方針
想定リスク:
- `vrc-loghook-gui` と同名の別実行ファイルが配置される
- 利用者が意図せず別GUIを起動してしまう

対策レベル（現行）:
- 誤起動検知を重視
- `--identity` が期待値でないGUIは起動拒否
- 期待ハッシュと不一致の場合は警告（`--gui-hash-warn=false` で抑止可能）

非目標（現行）:
- 同一ユーザー権限で実行される悪性GUIを完全に防ぐこと
- 署名検証のような強制的な改ざん防止

## 品質方針（Maintainer Quality Policy）
以下はプロジェクトとして、品質維持に努めます。
1. VRChat公式APIやVRChatアカウント認証を利用しない
2. ローカルログ駆動で動作する
3. 危険機能（hooks）を既定で無効にする
4. GUI起動時の識別チェックとハッシュ不一致警告を提供する
5. 設定項目・送信経路・制限事項を文書化する

## 非サポート範囲（Out of Scope）
以下はサポート対象外です。
1. 利用者端末が侵害されている状況での安全
2. 利用者が有効化した任意hookコマンドの安全性
3. 外部サービス（Discord等）側の障害・漏えい
4. 利用者が改変または非公式ビルドを使用した場合

今後の強化予定:
- OSごとの設定ファイル権限チェック強化
- リリース成果物の署名 / ハッシュ公開強化
- GUIレイヤーのセキュリティ表示改善
