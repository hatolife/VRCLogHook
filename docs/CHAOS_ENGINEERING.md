# VRC LogHook Chaos Engineering

この文書は、`VRC LogHook` の信頼性を継続的に検証するための小規模なカオスエンジニアリング手順です。  
参考: https://raw.githubusercontent.com/VoltAgent/awesome-claude-code-subagents/refs/heads/main/categories/04-quality-security/chaos-engineer.md

## 目的
- 想定外設定や運用ミスが起きても、監視が停止しないことを確認する。
- 失敗時の挙動（自動回復・フォールバック・ログ記録）を定期的に検証する。
- 実験結果を残し、改善アクションに結びつける。

## 対象と前提
- 対象: `core` の設定読込・ルール評価・監視ループの耐障害性
- 非対象: VRChat API連携（本プロジェクトでは未使用）
- 実験はローカル環境で実施し、実ユーザー影響を避ける。

## 実験チェックリスト
- 定常状態を定義したか（例: テスト成功、主要機能が継続稼働）
- 仮説を定義したか（例: 不正設定投入後も自動回復で起動継続）
- 影響範囲を限定したか（ローカル一時ディレクトリのみ）
- ロールバック可能か（テスト完了で自動破棄）
- 観測手段があるか（テストログ、終了コード）
- 学びと次アクションを記録したか

## 現在の実験セット
`scripts/chaos-local.sh` が以下を実行します。

1. 不正設定注入実験  
`LoadOrCreate` が不正設定を `.broken-*` に退避し、既定値で再生成することを確認。

2. ルール無効化実験  
`match.rules[].enabled=false` のルールがマッチ対象から除外されることを確認。

3. 高頻度リロード耐性実験  
ホットリロードが頻繁でも、監視処理が飢餓しないことを確認。

## 実行方法
```bash
./scripts/chaos-local.sh
```

## 期待される合格条件
- すべての実験テストが成功する（exit code 0）
- 失敗時にどのフェーズで落ちたかが標準出力で判別できる

## 運用ルール（推奨）
- リリース前に1回実行
- 監視・設定・通知周りの変更時は必須実行
- 失敗時は原因と再発防止策を `docs/AI_AUDIT.md` に追記
